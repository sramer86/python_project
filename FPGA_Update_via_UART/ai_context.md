请用 Python 编写一个 FPGA 串口在线升级上位机工具，要求如下：

1. **使用 tkinter 做 GUI 界面**，界面简洁清晰，符合工业工具风格。
2. 功能模块：
   - 串口配置区：串口号选择、波特率设置（可输入或下拉常见 9600/115200/230400/460800/921600）、打开/关闭串口按钮。
   - 烧录文件区：选择 .bin 文件路径、显示文件路径与大小。
   - 烧录配置区：输入 Flash 烧写起始地址（十六进制格式），32位地址使能复选框。选中后发送“使能4B地址”命令，取消选择发送“关闭4B地址”命令。
   - 操作区：【开始升级】按钮、【回读 Flash】按钮。
   - 进度与日志区：进度条实时显示烧录进度、文本框显示日志信息。
3. **烧录流程**
   - 点击【开始升级】后，读取选中的 .bin 文件。
   - 将.bin文件的大小除以4KB结果向上取整，例如文件大小1.351MB，除以4KB后向上取整就等于338，作为擦除扇区的计数。
   - 开始擦除已计算出的扇区擦除计数个sector，每次擦除前先发送“写使能”命令，再发送“擦除4KB”命令，烧写的起始地址为擦除的起始地址，每个擦除命令地址以此为基准递增4KB，每次发送完一个命令都要等待串口回传，如果接到的数据有连续5个0x66，则表示操作成功，如果接到的数据有连续5个0xEE，则表示发送错误，需要重新发送上一条命令，如果在100ms时间内没有收到任何有用数据，则退出当前流程，并报告超时断开警告，数据实时更新擦除进度条。
   - 准备烧写程序，每次读取.bin文件256个字节进行编程，每次编程前发送“写使能”命令，再发送“页编程”命令，以烧写的起始地址为编程的起始地址，每次递增256B地址，直致.bit文件结束，每次发送完一个命令都要等待串口回传，如果接到的数据有连续5个0x66，则表示操作成功，如果接到的数据有连续5个0xEE，则表示发送错误，需要重新发送上一条命令，如果在100ms时间内没有收到任何有用数据，则退出当前流程，并报告超时断开警告，数据实时跟新编程进度条。
   - 按照自定义串口升级协议发送数据，协议帧结构预见第5条。
   - 实时更新进度条（百分比），擦除和编程均显示一个进度条，只是名字不同。
   - 烧录完成后弹出提示框：“升级完成”。
   - 异常处理：超时、校验错误、串口断开等给出明确提示。
4. **回读 Flash 功能**
   - 输入FLASH容量（Mbit）。
   - 发送“读取”命令，从0地址开始读取，每次发完读取命令后fpga会按格式返回256个字节数据，计算校验和，如果校验和正确，地址+256，开始读取下一个地址，如果校验和错误，则重新发送当前地址，最多重复读取10次如果仍然错误则退出报校验错误，例如FLASH容量为4Mb，则从0地址开始读取，每次读取256B，总共有4Mbit/8=512KB，需要读取512KB/256B=2048次
   - 回读过程实时更新进度条
   - 将回读数据保存为 **.bin 文件**。
   - 计算并显示回读数据的 **校验和（计算CRC32）**。
5. **协议及命令格式**
   - 使能4B地址命令：帧头（55 66 99 AA）+ 命令（B7）+ 校验和（从命令字节开始累加，B7）+ 帧尾（AA 99 66 55）
   - 关闭4B地址命令：帧头 + 命令（E9）+ 校验和 + 帧尾
   - 写使能命令：帧头 + 命令（06）+ 校验和 + 帧尾
   - 擦除4KB命令：帧头 + 命令（20）+ 地址（以0x00800000为例，三字节 80 00 00 四字节 00 80 00 00）+ 校验和（从命令累加到地址）+ 帧尾
   - 页编程命令：帧头 + 命令（02）+ 地址（同上）+ 数据（最多256）+ 校验和（从命令累加到数据结束）+ 帧尾
   - 以上命令正确执行返回（66 66 66 66 66），数据错误返回（EE EE EE EE EE）
   - 读取命令：帧头 + 命令（03）+地址（同上）+ 校验和 + 帧尾
   - 读取返回数据格式：帧头（同上）+ 数据（固定256个字节）+ 校验和（数据累加和）+ 帧尾（同上）
6. **技术栈**
   - 使用 `pyserial` 进行串口通信
   - 使用 `tkinter` 做界面
   - 结构清晰、注释完整、可直接运行
   - 代码风格规范，便于后续扩展
